多镜像加速 — 为什么需要多个加速 URL？

由于 GFW 对 raw.githubusercontent.com 的封锁/限速，单一 CDN 源可能出现更新失败。 本项目提供 4 个独立 CDN 镜像，建议为同一规则同时添加 2–3 个镜像 URL， 当某个源失效时可自动切换备用源，确保订阅持续可用。

配置教程 — 如何导入加速订阅

uBlock Origin

  1. 右键 uBO 图标 → 仪表盘（Dashboard）
  2. 切换到 Filter lists 标签
  3. 展开 Custom 区域 → 点击 Import
  4. 粘贴加速 URL（一行一个）
  5. 点击 Apply changes → Update now

AdGuard 浏览器扩展

  1. 打开 AdGuard 设置页面
  2. 左侧菜单 → 过滤器
  3. 自定义过滤器 → 添加过滤器
  4. 粘贴加速 URL
  5. 等待订阅同步完成

AdGuard Home (DNS)

  1. 登录 AdGuard Home 管理面板
  2. Filters → DNS blocklists
  3. Add blocklist → 粘贴加速 URL
  4. Save & Update 强制刷新
  5. 建议配置上游 DNS（如 114.114.114.114）

性能建议

规则数量影响加载速度：浏览器扩展加载过多规则会占用内存并降低页面加载速度。建议采用"内置规则 + 2–4 条补充"的策略。

推荐组合示例：

  • 基础：EasyList + EasyPrivacy + AdGuard Base Filter（内置已含，无需重复添加）
  • 中国用户加：EasyList China + AdGuard Chinese Filter
  • 可选补充：乘风规则 或 Fanboy's Annoyances（二选一即可）
  • DNS 用户：StevenBlack 或 Hblock（AdGuard Home 专用）

更新频率：本项目通过 GitHub Actions 每 6 小时自动同步上游规则，确保时效性。

常见问题（FAQ）

更新失败怎么办？

1. 检查当前镜像是否可用 — 在浏览器中直接访问加速 URL 确认

2. 切换到其他镜像源 — 点击页面顶部"镜像"下拉框选择备用源

3. 为同一规则添加多个镜像 URL 实现冗余备份

4. AdGuard Home 用户：检查 Query Log 中的错误日志

规则不生效？

1. 确认已点击"Update now"或"Save & Update"强制刷新

2. 检查规则是否与已启用的内置规则冲突

3. 尝试清除扩展缓存后重新订阅

4. 确认加速 URL 可正常访问（非 404/超时）

如何检查更新状态？

1. 本页面顶部的"更新时间"徽章显示最近同步时间

2. uBlock Origin：Dashboard → Filter lists 查看各规则的上次更新时间

3. AdGuard Home：Filters 页面查看各订阅的同步状态

哪个镜像速度最快？

速度因地区和网络环境而异，建议：

1. 国内用户优先尝试 cosr 和 gitmirror

2. jsDelivr 全球覆盖好，国内偶尔波动

3. GHUCS 为 GitHub 官方 CDN，稳定性最高

4. 最佳方案：同时添加 2–3 个镜像，让工具自动选择可用源
